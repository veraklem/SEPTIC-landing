# ИНБИО-СЕПТ — лендинг

Имиджево-продающий лендинг биопрепарата **ИНБИО-СЕПТ** (разработано **INBIOLAB**, розничный бренд **INBIOSTORE**) для септиков и выгребных ям. Vite + React 18 + TypeScript + Tailwind CSS 3.4 + framer-motion. Backend отсутствует — сайт статический, «Купить»/«Получить консультацию» ведут на маркетплейсы (пока заглушки) и на `mailto`.

Директория проекта и имя npm-пакета (`inbiaccept`) сохранены как есть — это внутренний идентификатор, не пользовательский текст.

Полная дизайн-спецификация (цвета, типографика, тексты секций, анимации) — в [`SPEC.md`](./SPEC.md).

## Запуск

Требуется Node.js 18+.

```bash
npm install       # установка зависимостей
npm run dev       # локальный сервер разработки (Vite, с HMR)
npm run build     # проверка типов (tsc -b) + прод-сборка в dist/
npm run preview   # локальный просмотр собранного dist/
npm run lint      # ESLint
```

    products.ts   — форматы упаковки (карточки в секции «Форматы»)

## Где менять контент

Весь текстовый и «бизнесовый» контент вынесен из компонентов в `src/content/`:

- **`src/content/site.ts`** — название бренда/продукта, пункты меню навигации, контакты (email/телефон), список маркетплейсов (Ozon, Wildberries, Яндекс Маркет) с ссылками, и главный флаг:
  - `marketplacesLive: boolean` — переключатель «маркетплейсы уже доступны». Пока `false`: кнопки показывают состояние «скоро», не кликаются (`aria-disabled`). Когда появятся реальные ссылки в `marketplaces[].url` и нужно включить продажи — поставить `available: true` у нужной площадки и/или `marketplacesLive: true`.
- **`src/content/products.ts`** — карточки форматов упаковки (объём, изображение, характеристики, артикул, ссылка на сертификат). Сюда же добавлять новые объёмы, когда появятся.
- **`src/content/sections.ts`** — тексты и данные остальных секций: заголовки, подписи, шаги, преимущества, сравнение с химией и т.д.
- **`src/content/faq.ts`** — вопросы/ответы FAQ-аккордеона.
- **`src/content/privacy.ts`** — текст политики конфиденциальности, показанный в `PrivacyPolicyModal` (ссылка на неё — в футере).
- **`src/content/terms.ts`** - текст пользовательского соглашения, показанный в `TermsModal` (ссылка на него - в футере).
- **`src/content/law.ts`** - текст раздела «Правовая информация», показанный в `LawModal` (ссылка на него - в футере).

Компоненты в `src/components/` контент не хранят — они только рендерят то, что приходит из `content/*`. Поэтому для правки текста, ссылок или цифр обычно не нужно трогать `.tsx`-файлы вообще.

## Как заменить изображения

Все изображения сайта лежат в `public/images/` и подключены по абсолютным путям (`/images/...`) из `src/content/*`:

| Файл | Используется в |
|---|---|
| `hero-scene.webp` | фон-иллюстрация в Hero |
| `microbes-water.webp` | иллюстрация в TrustBlock |
| `septic-scheme.webp` | схема в HowItWorks («Как работает») |
| `product-bottle.webp` | флакон продукта (Hero, карточка формата в ProductFormats) |
| `og-cover.jpg` | Open Graph превью (шаринг ссылки) |
| `biotech-wave.webp` | фон декоративного разделителя `InteractiveWaveDivider` (между Benefits и UsageSteps) |

Чтобы заменить изображение — положите новый файл с тем же именем в `public/images/` (или новым именем + поправьте путь в соответствующем файле `src/content/*`). Компонент `ProductMockup` (`src/components/ui/ProductMockup.tsx`) умеет работать в двух режимах: `image` (если путь к картинке задан и валиден) и `illustration` (SVG-заглушка), поэтому подстановка нового рендера флакона не требует правок кода — только замены файла и/или пути в `src/content/products.ts` (`image: '/images/product-bottle.webp'`).

**Важно:** на текущем рендере флакона нанесён брендинг **INBIOSTORE / «ИНБИО-СЕПТ»**, который теперь совпадает с реальным продуктом — но само изображение всё ещё черновой макет-заглушка (не финальный дизайн упаковки). Перед публикацией сайта заменить `public/images/product-bottle.webp` на финальный рендер флакона **ИНБИО-СЕПТ**.

## Декоративный разделитель

`InteractiveWaveDivider` (`src/components/ui/InteractiveWaveDivider.tsx`) — полноширинный фоновый блок-разделитель между секциями: слоистая композиция из фото, размытых aqua-пятен и «пузырей» с лёгкой idle-анимацией и параллаксом по курсору. Без текста, чисто декоративный (`aria-hidden`), в разметке используется один раз — между `Benefits` и `UsageSteps` в `src/App.tsx`.

Пропсы:

| Проп | Тип | По умолчанию | Назначение |
|---|---|---|---|
| `src` | `string` | `/images/biotech-wave.webp` | путь к фоновому изображению |
| `height` | `number` | — | высота блока в px (иначе адаптивная `150/200/240px`) |
| `opacity` | `number` | `1` | прозрачность фото-слоя |
| `intensity` | `number` | `1` | сила курсорного параллакса |
| `className` | `string` | — | доп. классы контейнера |

Пример вставки между секциями:

```tsx
import { InteractiveWaveDivider } from './components/ui/InteractiveWaveDivider'

<Benefits />
<InteractiveWaveDivider height={220} intensity={1.2} />
<UsageSteps />
```

Изображение меняется либо заменой файла `public/images/biotech-wave.webp`, либо через проп `src`. На устройствах без точного курсора (`hover: none` / тач) и при `prefers-reduced-motion` параллакс и idle-анимация отключаются автоматически.

## Финальный блок (CtaFooter): фон и контакты

Последняя секция сайта (`#contact`, компонент `src/components/sections/CtaFooter.tsx`) - CTA + панель прямых контактов на фоновом изображении + светлый footer-стеклобар (`FooterLinks`). Заменяет прежние `FinalCTA` и `Footer`.

### Замена фонового изображения

- Изображение: `public/images/water-flower-bg.webp` (полноэкранный фон секции, `object-cover`).

Чтобы заменить - положить файл с тем же именем по тому же пути (или новое имя + поправить `src` у `<img>` в `CtaFooter.tsx`).

### Сбор заявок отключён - связь через прямые каналы

Сайт **не собирает и не обрабатывает** персональные данные посетителей: формы обратной связи на сайте нет. Вместо неё в `CtaFooter.tsx` - панель прямых ссылок на внешние каналы (Telegram-бот, почта, Telegram-канал), переход по которым не создаёт обработки данных на стороне сайта. Список каналов и подписи редактируются прямо в `DIRECT_CHANNELS` в начале `CtaFooter.tsx`; e-mail берётся из `site.contacts.email` (`src/content/site.ts`).

Если в будущем понадобится снова собирать заявки через сайт - потребуется вернуть форму (была реализована в `src/components/sections/ContactForm.tsx`, удалена по решению клиента) и переменную окружения для приёма данных; шаблон переменных окружения - в [`.env.example`](./.env.example) в корне проекта.
